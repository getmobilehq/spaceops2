01

Product overview

SpaceOps is a facility management platform for commercial cleaning operations. It handles scheduling, GPS-verified attendance, quality inspections, issue tracking, reporting, payroll, and client transparency.

Tech Stack

Frontend: Next.js 14 (React), TailwindCSS, shadcn/ui components

Backend: Supabase (PostgreSQL + Auth + Storage + RLS)

Payments: Stripe (subscriptions, checkout, customer portal)

AI: Anthropic Claude (floor plan detection, report summaries)

Languages: English, Spanish, French (auto-detected, switchable)

Access Method

Responsive web application. No native mobile app. Janitors use mobile browsers. Supervisors and admins use desktop or tablet. GPS and camera access are requested via browser APIs for clock-in verification and QR scanning.

5.5 Payroll

Calculation Formula

Supervisor flows

Day-to-day operations — scheduling, inspections, attendance, and issue management, scoped to assigned buildings.

6.1 Activities

  • Create activity: floor, name, date, time window, notes
  • Room tasks auto-created for all active rooms on selected floor
  • Assign janitors to room tasks individually or in bulk
  • Publish activity → visible to janitors in “Today”
  • Cancel, close, and delete (draft only) activities
  • Room task inspection: mark pass/fail with notes

6.2 Inspections

  • Create inspection for room in assigned building
  • QR scanning opens inspection checklist for correct room
  • Checklist items: completed + optional notes + optional photos
  • Room must be scanned before completing inspection
  • Pass rate calculated from checklist vs. org threshold
  • Failed inspection can trigger deficiency creation

6.3 Activity Templates

  • Create template: name, floor, time window, optional recurring config
  • Recurring: select days (Mon–Sun), time slots, preset (daily, twice daily, etc.)
  • Recurring templates auto-generate activities
  • Toggle recurring active/inactive
  • Save existing activity as template

6.4 One-Off Tasks

  • Create: title (1–200), description, due date/time, assign to janitor, optional image
  • Appears in janitor’s “Today” view on due date
  • Janitor can mark as done

6.5 Issues & Deficiencies

  • Create: description (1–500), severity (low/medium/high), optional assignee
  • Status workflow: open → in progress → resolved
  • SLA timers: high 24h, medium 48h, low 168h
  • Visible in client portal

6.6 Attendance & Reports

  • Today’s attendance for assigned buildings (read-only)
  • Reports: same as admin but scoped to assigned buildings

Janitor flows

Mobile-first task execution — clock in, complete rooms, report issues, clock out.

7.1 Clock In

  • Must clock in before seeing tasks
  • GPS coordinates captured at clock-in
  • Within geofence (default 150m) = • Verified
  • Outside geofence = • Unverified (still allowed)
  • GPS permission denied: clock-in works, geo_verified = false
  • Cannot clock in twice same day same building

7.2 Today’s Tasks

  • Shows only today’s active tasks and ad-hoc tasks
  • Tap room → checklist items (optional notes + photos)
  • Mark task “Done” or “Has Issues”

7.3 Issue Reporting

  • Report from within a task or standalone
  • Fields: room, description (1–500), severity
  • Supervisor is notified

7.4 Clock Out & History

  • Clock-out records timestamp, duration calculated
  • Hours feed into reports and payroll
  • Attendance history shows past records with GPS status
08

Client flows

Read-only transparency dashboard for building owners.

  • Overview: buildings, pass rates, SLA compliance (on-track / at-risk / breached)
  • Activities: cleaning history with pass rates (read-only)
  • Issues: open/in-progress/resolved deficiencies (read-only)
  • Cannot create, edit, or delete any resources
  • Cannot see other clients’ buildings or data
  • Cannot see payroll, billing, or user management
09

Platform admin flows

Super admin cross-organisation management. Requires is_super_admin = true.

  • Orgs list: all organisations with name, plan, created date
  • Change plan (free/pro/enterprise) with audit log
  • Suspend org with reason — users blocked from accessing
  • Unsuspend org — access restored
  • Delete org (type name to confirm) — cascades all data
  • Org disappears from list immediately after deletion
  • All actions logged in platform audit trail
  • Usage stats: AI vectorisation, AI reports, API calls by org

13

Security test scenarios

Organisation Isolation

  • User in Org A cannot see buildings, users, or data from Org B
  • Manually crafting URLs with another org’s slug returns 404

Role-Based Access

  • Janitor cannot access /{org}/admin/*
  • Client cannot access supervisor routes
  • Supervisor can only see assigned buildings
  • Janitor can only see their own tasks

Email Enumeration Prevention

  • Password reset always shows success regardless of email validity
  • Registration error for duplicate email is generic

Session & Account Security

  • Expired sessions redirect to login
  • Deactivated users are signed out and blocked
  • Suspended org users cannot access the app

Data Integrity

  • Deleting building cascades to floors, rooms, and related data
  • Deleting client unlinks (does not delete) buildings
  • Approved payroll runs cannot be deleted
  • Closed/cancelled activities cannot be edited

Recommended test paths

Path A — Admin Full Onboarding High Priority
  1. Register a new organisation
  2. Complete onboarding wizard (building + invite)
  3. Add floors and rooms to the building
  4. Upload floor plan and run AI detection (Pro plan)
  5. Create checklist template and set as default
  6. Invite a supervisor, janitor, and client
  7. Assign supervisor to the building
  8. Update settings (pass threshold, logo)
  9. View billing and upgrade plan
  10. Generate payroll run and approve
Path B — Supervisor Daily Operations High Priority
  1. Log in as supervisor
  2. Create an activity for today
  3. Assign janitors to rooms
  4. Publish the activity
  5. Create a one-off ad-hoc task
  6. Run an inspection (scan QR, complete checklist)
  7. Record a pass and a fail
  8. Create a deficiency from the failed inspection
  9. View attendance dashboard
  10. Generate a report
Path C — Janitor Shift High Priority
  1. Scan building QR code to clock in
  2. Verify GPS verification status
  3. View today’s tasks
  4. Scan room QR to start a task
  5. Complete checklist items (with photos on required items)
  6. Mark task as done
  7. Report an issue on a different room
  8. Complete an ad-hoc task
  9. Clock out
  10. View attendance history
Path D — Client Portal Medium Priority
  1. Log in as client
  2. View building overview and SLA metrics
  3. View recent activities and pass rates
  4. View open issues
  5. Confirm client cannot create or modify anything
  6. Confirm client cannot see other clients’ data
Path E — Payroll End-to-End Medium Priority
  1. Set pay rates for 3 employees (different rates and thresholds)
  2. Ensure attendance records exist for a date range
  3. Generate payroll run for that period
  4. Verify calculations: regular hours, overtime, gross pay
  5. Verify employee with no attendance shows $0
  6. Export CSV and verify data matches UI
  7. Approve the run
  8. Confirm approved run cannot be deleted
  9. Generate another draft run and delete it
Path F — Platform Admin Low Priority
  1. Log in as super admin
  2. View orgs list
  3. Change an org’s plan
  4. Suspend an org — verify its users cannot log in
  5. Unsuspend — verify access restored
  6. Delete a test org — verify data removed
  7. View usage stats
Path G — Edge Cases & Negative Testing Medium Priority
  1. Submit all forms with empty required fields — verify validation messages
  2. Submit with values exceeding max lengths
  3. Attempt to access another org’s data via URL manipulation
  4. Upload oversized files (>10 MB floor plan, >2 MB logo)
  5. Upload unsupported file types
  6. Clock in twice on the same day — verify error
  7. Try to delete an approved payroll run — verify blocked
  8. Try to edit a closed activity — verify blocked
  9. Navigate as a deactivated user — verify redirect to login
  10. Test in Spanish and French — verify all labels translate
